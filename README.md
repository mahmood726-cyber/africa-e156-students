## What is E156?

E156 is a micro-paper format: exactly **7 sentences**, exactly **156 words**, in a single paragraph. Each sentence has a fixed role:

| # | Role | ~Words |
|---|------|--------|
| S1 | Question | 22 |
| S2 | Dataset | 20 |
| S3 | Method | 20 |
| S4 | Primary Result | 30 |
| S5 | Robustness | 22 |
| S6 | Interpretation | 22 |
| S7 | Boundary/Limitation | 20 |

## Your Assignment

1. **Read** the AI-drafted E156 paper
2. **Study** the interactive HTML dashboard
3. **Download** and review the Python analysis code
4. **Rewrite** the paper in your own voice (keep the 7-sentence, 156-word format)
5. **Add 2-3 references** (PubMed, WHO, ClinicalTrials.gov)
6. **Include GitHub links** to the dashboard and code in your Outside Note Block
7. **Submit** to [Synthesis Medicine Journal](https://www.synthesis-medicine.org/index.php/journal)

**Important:** These papers are AI-generated drafts. You must rewrite them substantially. The journal will check for originality. Your perspective as a Ugandan researcher adds real value.

## Running the Code

### Requirements

- Python 3.8+
- `requests` package: `pip install requests`

### Scripts

**Country analysis** (queries ClinicalTrials.gov API — no key needed):
```bash
cd analysis
python fetch_africa_rcts_by_country.py
```

**Statistical deep dive** (runs on cached data from above):
```bash
cd analysis
python statistical_deep_dive.py
```

**Individual paper scripts** (in each group's `code/` folder):
```bash
cd geographic-equity/code
python angle-11-city-dispersion-rates.py
```

Most scripts query the ClinicalTrials.gov API v2 directly (public, no API key required). A few scripts need data generated by other scripts first — check the script's header comments.

## Data Source

All data comes from **ClinicalTrials.gov API v2** (https://clinicaltrials.gov/data-api/api), a public registry maintained by the U.S. National Library of Medicine. No API key is required.
